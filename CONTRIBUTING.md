# Contributing to Playwright Case Studio

Read `README.md`, `ABOUT.md`, and `ROADMAP.md` before contributing.

This project is intended for authorized software testing. Contributions should improve test-case editing, reproducibility, diagnostics, accessibility, or documented export behavior.

## Contribution areas

- Project and test-case data models.
- Step editing, ordering, grouping, and assertions.
- Browser and device-profile configuration.
- Test-run status, screenshots, traces, and reports.
- Import and export fixtures.
- Accessibility, tests, packaging, and documentation.

Use an Issue before changing the case schema, runner contract, report format, or export layout.

Pull requests should include the testing problem, intended behavior, non-goals, supported environment, validation steps, fixture input, expected result, and compatibility impact.

Use focused Conventional Commit messages such as `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, and `chore:`.

Contributors are responsible for correctness, licensing, and review. Do not include private websites, real user data, or testing material you are not permitted to publish.
